Pandorica Graphics

What is Pandorica Graphics?

Pandorica Graphics is a WIP graphics engine optimized for speed.
It supports 3D and 2D, however everything in the engine is a 2D surface (plain). In the 3D context, these plains can be turned, scaled and moved around.
The engine is highly customizable and works out of the box. To modify the engine to you liking, you would not use the predefined "main"-functions and instead develop your own.
Pandorica Graphics is very low-level and is directly using vulkan, directX, metal, etc. through SDL3 and is thus compatible with many platforms.

Getting started

Requirements

SDL3, SDL3_image, SDL3_ttf, SDL3_mixer and optionally SDL3_shadercross.

Installing

Download the binaries and after that include the files in your project.

How it works

Pandorica Graphics consists of 2 types: devices and resources.
Devices can only exist once per programm and have to be initialized to use. These devices then form a 'context'. Subordinated to these devices are the resources. They can be objects, textures, shaders and many more. Consult the documentation (or the header file of the specific type) for more information.

How to use it

To use Pandorica Graphics the following steps have to be done pretty much all the time:

  • Initialize the context with 'pg::Init()'.
  • Create window and GPU with 'pg::CreateWindowAndGPU()'.
  • If you plan to use the built-in rendering process run 'pg::CreateStandardContext()'.
